# Architect - アプリ設計支援AI ユーザーガイド

## アクセス情報

**目的**: ユーザーの曖昧なアイデアを、わずか5分の対話でManus1.5で実装可能な完璧な仕様書に変換します。

**アクセス**: ログインが必要です。Manus OAuthでログインできます。

## Powered by Manus

Architectは最先端の技術スタックで構築されています。

**フロントエンド**: React 19 + TypeScript + Tailwind CSS 4 + shadcn/ui による高速でモダンなユーザーインターフェース。レスポンシブデザインで、あらゆるデバイスに最適化されています。

**バックエンド**: Express 4 + tRPC 11 による型安全なAPI。エンドツーエンドの型推論により、開発効率とコード品質を最大化しています。

**データベース**: MySQL/TiDB + Drizzle ORM による高速で信頼性の高いデータ永続化。会話履歴と生成された仕様書を安全に保存します。

**AI機能**: 最新のLLMを活用した対話型ヒアリングと仕様書自動生成。ユーザーの意図を深く理解し、実装可能な設計書を生成します。

**認証**: Manus OAuth による安全でシームレスな認証フロー。

**デプロイ**: 自動スケーリングインフラストラクチャとグローバルCDNにより、世界中どこからでも高速アクセスが可能です。

## ウェブサイトの使い方

Architectは5段階のフェーズで、あなたのアイデアを完璧な仕様書に変換します。

ログイン後、チャット画面が表示されます。「どんなアプリを作りたいですか?」という質問から始まります。ざっくりとしたアイデアを入力してください。例えば「消える手紙アプリ」のように簡潔で構いません。

AIが質問を重ねながら、アイデアの核心、ターゲットユーザー、コア機能、デザインの雰囲気、技術的制約を順番に明確にしていきます。画面上部のプログレスバーで、現在どのフェーズにいるかが一目でわかります。

各フェーズで、AIは選択肢を提示したり、類似アプリの例を挙げたりして、あなたの回答をサポートします。右側のプレビューエリアには、収集した情報がリアルタイムで表示されます。

全5フェーズが完了すると、「仕様書を生成」ボタンが有効になります。このボタンをクリックすると、AIが完全な設計書とManusプロンプトを自動生成します。

生成された仕様書ページでは、アプリ概要、機能一覧、画面遷移図、データベース設計、技術要件、デザイン要件などが整理されて表示されます。最も重要なのは、ページ下部に表示されるManusプロンプトです。「コピー」ボタンをクリックして、このプロンプトをManus1.5に貼り付けるだけで、アプリの実装が始まります。

「履歴」ボタンから、過去の会話や生成した仕様書を確認できます。途中で離脱しても、後から続きを再開できます。

## ウェブサイトの管理

**設定パネル**: 管理UIの「Settings」から、ウェブサイト名やロゴをカスタマイズできます。

**データベース管理**: 管理UIの「Database」パネルから、会話履歴や仕様書のデータを直接確認・編集できます。

**ダッシュボード**: 管理UIの「Dashboard」で、アクセス統計や利用状況を確認できます。

## 次のステップ

Manusに話しかけて、いつでも機能の追加や変更をリクエストできます。

生成されたManusプロンプトをコピーして、Manus1.5に貼り付けてください。あなたのアイデアが、数分で動くアプリに変わります。
